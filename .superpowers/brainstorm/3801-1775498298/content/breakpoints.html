Section 1: Breakpoint System

Four size classes based on logical width — covers every device from small phone to large tablet

xs
< 480px
small phone
portrait
sm
480 – 720px
large phone
portrait / small landscape
md
720 – 1024px
tablet portrait
phone landscape
lg
> 1024px
tablet landscape
large display

Scale tokens per breakpoint

Every dimension derived from a single scale factor — nothing hardcoded

Token xs sm md lg
scaleFactor 0.80 0.90 1.00 1.20
spriteScale (grid) 4.0 5.0 6.0 8.0
gridMaxExtent 160px 200px 240px 300px
buttonSize 44px 48px 56px 64px
basePadding 12px 16px 20px 28px

AppResponsive API (expanded)

AppResponsive.sizeClass(context) // → SizeClass.xs/sm/md/lg
AppResponsive.scaleFactor(context) // → 0.80–1.20
AppResponsive.spacing(context, base) // → base × scaleFactor
AppResponsive.fontSize(context, base) // → base × scaleFactor
AppResponsive.iconSize(context, base) // → base × scaleFactor
AppResponsive.buttonSize(context) // → 44–64 by size class
AppResponsive.spriteScale(context) // → 4.0–8.0 by size class
AppResponsive.gridMaxExtent(context) // → 160–300 by size class
AppResponsive.isCompact(context) // → true if xs or sm
AppResponsive.isPortrait(context) // → orientation check
