Kid Home Screen — Portrait Layout Options

How should the card grid behave in portrait on a phone?

A — Grid Reflows

Same card grid, fewer columns in portrait. 2 columns on phones, 3–4 on tablets. Cards are bigger in portrait — great for small fingers. Minimal code change.

B — List in Portrait

Portrait shows a horizontal-card list (sprite + title). Landscape stays as the grid. More content visible at once, but loses the "tap a big picture" feel that kids love.

C — Size-Adaptive Grid

Column count driven purely by available width. 2 cols on small portrait phones, 3 on larger phones, 4–5 on tablets. Same card component everywhere — only the count changes. Most robust for "any device."
