Section 2: Screen-by-Screen Changes

What changes on each screen — and what stays the same

main.dart — Unlock orientation

SystemChrome.setPreferredOrientations([landscapeLeft, landscapeRight])
SystemChrome.setPreferredOrientations([]) // all orientations

One line change. Everything else in main.dart stays (immersiveSticky, wakelock).

KidHomeScreen — Grid + Player Pill

Phone portrait (xs)
2 cols · sprite 4.0 · pill full-width
Tablet landscape (lg)
4+ cols · sprite 8.0 · pill floating pill

Grid: maxCrossAxisExtent from AppResponsive.gridMaxExtent() — no column count to manage manually. Player pill: full-width bar in portrait (xs/sm), floating pill in landscape (md/lg) — same widget, different constraints.

ParentEditScreen — Orientation-aware layout

Implemented with AppResponsive.isPortrait(context) → switches between Column and Row layout. No separate widget trees.

Other screens — minimal changes

Screen Change
